# Perceval Composer — Final Fixes Walkthrough

I have addressed all the issues you encountered when trying to launch and run the project. The Perceval Composer is now fully functional and verified end-to-end!

## What was fixed

1. **PowerShell `start.bat` Issue**
   The error `The term 'start.bat' is not recognized` was caused by PowerShell's strict security policy. In PowerShell, you must prefix local executable files with `.\`. 
   * **Fix**: Run `.\launcher\start.bat` or just `.\start.bat` from the launcher directory. Alternatively, double-clicking the `start.bat` file in Windows Explorer will work perfectly.

2. **Backend Startup Crash (`ModuleNotFoundError`)**
   When `start.bat` ran `uvicorn backend.main:app`, the backend was crashing silently because `main.py` tried to do an absolute import of `perceval_runner`, which wasn't in the system path.
   * **Fix**: Changed the import in `backend/main.py` to a relative import (`from .perceval_runner import run_perceval_code`).

3. **Perceval 1.1.0 Installation**
   The `requirements.txt` file was previously pointing to an older version `0.11.0`, but we upgraded to `1.1.0` in the last conversation to fix API compatibility.
   * **Fix**: Updated `requirements.txt` to specify `perceval-quandela==1.1.0` so that `start.bat` properly installs the correct version automatically.

4. **Code Generator Unicode Error**
   When you ran the code generated by the frontend, the backend would crash due to a Windows `cp1252` console encoding error when trying to print the `⟩` character.
   * **Fix**: Updated the JavaScript code generator in `frontend/src/utils/codeGenerator.js` to replace the unicode bracket `⟩` with a standard ASCII `>` character, matching the fix we applied to the CHSH demo script previously.
